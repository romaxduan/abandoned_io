This is the driver module.

Description

This is the driver module. Driver module is the one which does the initialization and starts the simulation. It creates two ets:named_table for clients and server to keep their keys info. It will read the keys from local key files.

Function Details

read_key/1

read_key(Name::string()) -> RSAKey

Read the key from pem files. This function takes filename as parameter and read the file to get the content. Then it will use built-in public_key module to get the key entries and then decode as PublicKey or PrivateKey.

start_ns/0

start_ns() -> ok

Start the NS Protocol simulation. This function will setup the server_table and key_table, spawn two clients and one server, then ask one of the clients (Alice) to initiate a session with the other client (Bob). It will also print the start time and end time using the formats of Erlang:now().

start_nsl/0

start_nsl() -> ok

Start the NS-Lowe Protocol simulation. This function will setup the server_table and key_table, spawn two clients of clients_lowe and one server, then ask one of the clients (Alice) to initiate a session with the other client (Bob). It will also print the start time and end time using the formats of Erlang:now(). It's the improved version and will prevent man-in-the-middle attack.
